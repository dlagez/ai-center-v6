# 1. 项目背景

当前业务中存在一类典型操作场景：
业务人员需要定期从内部系统中获取项目数据，并将相关数据回填至既有 Excel 台账中，例如清欠表、项目台账、收付款统计表等。

现阶段主要存在以下问题：

1. **人工处理量大**
   需要人工从系统中导出数据，再逐条对照 Excel 项目编号进行填报，耗时长、重复性高。

2. **容易出错**
   Excel 中项目较多、月份列较多、字段结构复杂，人工更新容易发生行列写错、漏填、覆盖旧值不留痕等问题。

3. **缺乏审计追踪**
   当前更新过程大多缺乏完整日志，无法清晰追溯“谁在什么时间更新了哪些项目、覆盖了哪些值”。

4. **复用性差**
   每次都依赖个人经验处理，无法形成统一服务能力，后续扩展到“计量产值、回款、累计值”等场景成本高。

因此，拟建设一套**Excel 自动更新服务**，通过调用内部业务 API 获取业务数据，根据 Excel 表格中的**项目编号**进行精准匹配，并将返回的多条数据批量更新到 Excel。
同时，引入大模型作为任务理解和结果说明层，以提升系统的人机交互能力与易用性。

---

# 2. 建设目标

本项目目标是建设一套**可复用、可审计、可扩展**的 Excel 自动更新服务，实现以下能力：

1. 支持上传 Excel 文件并解析目标工作表；
2. 支持从内部系统 API 获取项目数据；
3. 支持基于“项目编号”匹配 Excel 中对应项目行；
4. 支持将业务系统返回的多条数据按匹配规则批量写入 Excel；
5. 支持输出更新结果、未匹配清单、异常清单、变更日志；
6. 支持后续扩展到多种查询条件、多模板、多业务来源；
7. 支持通过大模型解析自然语言任务指令，并生成更新结果说明。

---

# 3. 产品定位

本产品定位为：

**“规则驱动的 Excel 更新执行服务 + 大模型辅助理解与说明能力”**

说明如下：

* **规则服务**负责：

  * 调业务 API
  * 解析 Excel
  * 行列定位
  * 精准写值
  * 日志记录
  * 审计追踪

* **大模型**负责：

  * 理解用户自然语言指令
  * 提取任务参数
  * 生成任务执行说明
  * 生成异常总结与人工确认建议

本产品**不采用“大模型直接操作 Excel 单元格”的模式**，而采用“模型理解 + 工具调用 + 确定性执行”的架构，以保证稳定性和准确性。

# 4. 适用场景

一期重点支持以下场景：

## 4.1 核心场景

根据业务系统返回的批量项目数据，将系统中的多条数据按项目编号自动更新到 Excel 表格中的对应位置。

例如：

* 根据接口返回结果批量更新项目清欠表中的对应值
* 根据筛选条件获取数据后批量回填 Excel 台账
* 根据项目编号将多条业务数据同步到指定工作表

# 5. 范围说明

一期纳入范围：

# 9. 功能需求

---

## 9.1 功能一：上传 Excel 文件

### 功能描述

用户可上传待更新的 Excel 文件，系统完成文件存储和基础校验。

### 输入

* Excel 文件（`.xlsx`）

### 处理逻辑

* 校验文件格式；
* 记录文件上传信息；
* 存储原始文件；
* 返回文件 ID。

### 输出

* 文件 ID
* 文件名
* 上传时间
* 文件状态

### 校验规则

* 仅支持 `.xlsx`
* 文件大小限制由运维配置，默认不超过 20MB
* 文件不得损坏

---

## 9.3 功能三：调用业务 API 获取项目数据

### 功能描述

系统根据查询条件从内部 API 获取标准化项目数据，用于后续批量更新 Excel。

### 输入

* 查询条件
* 可选筛选条件
* 映射所需字段

### 标准返回结构

```json
[
  {
    "project_no": "HXZC--2025.03.05-0002",
    "project_name": "示例项目",
    "date": "2026-03",
    "actual_output": 26.50,
    "metered_output": 20.00,
    "received_payment": 15.00
  }
]
```

---

## 9.4 功能四：Excel 模板解析

### 功能描述

系统自动识别 Excel 中的表头结构、项目编号列以及需要回填的目标列。

### 处理内容

* 识别表头区域；
* 识别多级表头；
* 识别“项目编号”所在列；
* 识别需要回填的业务字段对应列。

---

## 9.6 功能六：批量写入 Excel

### 功能描述

系统将 API 返回的多条数据按匹配规则写入 Excel 对应项目行的目标列。

### 支持字段

一期可优先支持：

* `actual_output`（实际产值）
* `metered_output`（计量产值）
* `received_payment`（回款）


### 写入规则

* 逐条遍历业务系统返回的数据；
* 根据匹配键定位 Excel 对应行；
* 根据字段映射定位目标列；
* 写入前记录旧值；
* 写入后记录新值；
* 保留原 Excel 单元格格式；
* 不改变非目标单元格内容。

---

## 9.7 功能七：输出更新结果文件

### 功能描述

更新完成后，系统生成新的 Excel 文件供用户下载。

### 输出内容

* 更新后的 Excel 文件
* 文件名建议包含时间戳
* 原文件不覆盖

### 命名建议

`原文件名_更新后_yyyyMMddHHmmss.xlsx`

---

## 9.8 功能八：生成更新日志与异常清单

### 功能描述

系统输出本次任务的执行结果明细。

### 输出内容

#### 更新结果汇总

* 总项目数
* 成功更新数
* 未匹配数
* 重复编号数
* 空值跳过数
* 覆盖写入数

#### 异常清单

* 未匹配项目编号
* 重复项目编号
* 缺失目标列
* 数值格式异常
* 写入失败记录

#### 变更日志

* 项目编号
* 行号
* 列号
* 原值
* 新值
* 写入时间
* 操作人 / 任务 ID

---

## 9.9 功能九：自然语言任务解析（AI 辅助）

### 功能描述

系统支持用户通过自然语言下达简单更新任务。

### 示例输入

* 用项目系统返回的数据更新这份表
* 把查询结果批量同步进这个表
* 更新软件项目清欠表中的项目数据

### 结构化输出示例

```json
{
  "sheet_name": "软件项目清欠表-收付款",
  "match_key": "project_no",
  "query_conditions": {
    "month": "2026-03",
    "field": "actual_output"
  },
  "update_mode": "compare_then_overwrite"
}
```

### 说明

AI 不直接修改 Excel，只负责参数理解与结果说明。

# 14. 异常处理

系统需支持以下异常类型：

1. 文件格式错误
2. 文件损坏
3. 工作表不存在
4. 缺少项目编号列
5. 缺少目标列
6. 缺少字段映射关系
7. API 调用失败
8. API 返回数据为空
9. 项目编号为空
10. 项目编号重复
11. 数值格式异常
12. 单元格写入失败

异常返回原则：

* 明确错误类型
* 明确影响范围
* 明确是否允许继续执行
* 允许导出异常清单
